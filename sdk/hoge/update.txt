ディレクトリのアップロードの仕方

mkdir ディレクトリ名   //「ディレクトリ名」のディレクトリを作成

～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～

git add .

git commit -m "メッセージ"

git push origin main //ここでアップデートされる


～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～～

git add . は 現在のディレクトリ内のすべての変更をステージングする コマンドです。
Gitでは、変更をコミットする前に「ステージング」する必要があります。
このコマンドを使うと、以下のような変更が対象になります：
- 追加されたファイル（新しく作成されたファイル）
- 変更されたファイル（内容が更新されたファイル）
- 削除されたファイル（削除が記録される）
git add . の . は 現在のディレクトリとそのサブディレクトリ全体 を指します。つまり、このコマンドを実行すると、カレントディレクトリ以下のすべての変更がステージングされます。

もし特定のファイルだけをステージングしたい場合は、
以下のように書きます：

git add ディレクトリ名

＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

git commit -m "<メッセージ>" は 変更をコミット（保存）するためのコマンド です。
構造：
- git commit → ステージングされた変更をコミット（保存）する
- -m "<メッセージ>" → コミットに説明メッセージを付ける（-m は "message" の略）

＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

git push origin main はローカルの main ブランチの変更をリモートリポジトリ (origin) に送信するコマンドです。

構造：
- git push → ローカルの変更をリモートリポジトリへ送信
- origin → リモートリポジトリの名前（通常、GitHubやGitLabのリポジトリ）
- main → プッシュするブランチ名（通常はメインの開発ブランチ）

例えば、ローカルで作業してコミットした後にこのコマンドを実行すると、リモートリポジトリの main ブランチが更新され、他のチームメンバーが最新の変更を取得できるようになります。

💡 **注意点**
- リモートにすでに更新された main ブランチがある場合、直接プッシュするとエラーが発生することがあります。
その場合は git pull --rebase で同期してからプッシュしましょう。
- もし main ブランチではなく別のブランチをプッシュしたいなら、
例えば git push origin feature-branch のようにブランチ名を変更すればOKです。
